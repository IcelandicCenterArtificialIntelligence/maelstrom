Meet Maelstrom

Maelstrom packages your Rust tests into hermetic micro-containers, then schedules and distributes them amongst an arbitrarily large cluster of test-runners. Currently supporting RUST. Typescript, python, and Java coming soon.

Maelstrom functions as a drop-in replacement for cargo-test. It will work out of the gate for most projects, though some slight configuration might be needed for more complex environments.
Maelstrom runs every test in its own container, hermetically, so no more "well it works on my machine." No more unintended dependencies on the testing environment, or unintended testing interactions.
Each test is run independently, in its own environment which increases reliability and reduces test interdependencies.
By adding more worker machines you can linearly improve test throughput. You can add lots of nodes to make your tests go faster.
Workers are rootless for improved security, and you can run it anywehere.
Maelstrom has its own container implementation, not relying on docker or RunC, and our containers are optimized to start quickly be low-overhead.
The whole project is created, from the ground up in Rust.
