Meet Maelstrom

Maelstrom packages your Rust tests into hermetic micro-containers, then distributes them to be run on an arbitrarily large cluster of test-runners.

  • It's easy. Maelstrom functions as a drop-in replacement for cargo-test, so for most projects, it will work right out of the gate.
  • It's reliable. Maelstrom runs every test hermetically, in its own container, and runs each test independently. Never deal with unintended dependencies on the testing environment, or problematic testing interactions again.
  • It's scalable. By adding more worker machines you can linearly improve test throughput. You can add nodes arbitrarily to make your tests go faster.
  • It works everywhere. Workers are rootless containers for improved security, and you can run them anywhere.
  • It's clean. Maelstrom has its own container implementation, not relying on docker or RunC, and our containers are optimized to start quickly be low-overhead.
  • It's Rusty. The whole project is created, from the ground up in Rust.
Maelstrom is currently available for Rust on Linux. Typescript, Python, and Java coming soon.
